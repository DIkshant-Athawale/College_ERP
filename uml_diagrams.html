Sequence Diagram — Login Flow

Sequence 
sequenceDiagram
    actor User as 👤 User (Browser)
    participant LP as Login Page
    participant AC as AuthContext
    participant API as API Layer (Axios)
    participant BE as Backend (auth.js)
    participant DB as MySQL DB

    User->>LP: Enter email + password
    LP->>AC: authApi.login({ email, password })
    AC->>API: POST /login
    API->>BE: POST /login { email, password }

    BE->>DB: SELECT * FROM teachers WHERE email = ?
    alt Teacher Found
        DB-->>BE: Teacher record
        BE->>BE: bcrypt.compare(password, hash)
        BE->>BE: Generate accessToken (JWT, 30m)
        BE->>BE: Generate refreshToken (JWT, 7d)
    else Teacher Not Found
        DB-->>BE: No result
        BE->>DB: SELECT * FROM students WHERE email = ?
        DB-->>BE: Student record
        BE->>BE: bcrypt.compare(password, hash)
        BE->>BE: Generate accessToken (JWT, 30m)
        BE->>BE: Generate refreshToken (JWT, 7d)
    end

    BE->>DB: DELETE FROM refresh_tokens WHERE user_id = ?
    BE->>DB: INSERT INTO refresh_tokens (token, user_id, ...)
    DB-->>BE: OK

    BE-->>API: { accessToken, role, userType } + Set-Cookie: refreshToken (HttpOnly)
    API-->>AC: Response data
    AC->>AC: Store token in localStorage
    AC->>AC: Set user state

    alt Role = admin
        AC-->>LP: Redirect → /admin
    else Role = student
        AC-->>LP: Redirect → /student/dashboard
    else Role = faculty
        AC-->>LP: Redirect → /teacher/dashboard
    end

Sequence Diagram — Token Refresh Flow

Sequence 
sequenceDiagram
    participant Client as 🖥️ Client (Axios)
    participant Interceptor as Axios Interceptor
    participant BE as Backend (auth.js)
    participant DB as MySQL DB

    Client->>BE: Original API Request (expired token)
    BE-->>Client: 401 Unauthorized

    Client->>Interceptor: Response interceptor detects 401
    Interceptor->>BE: POST /login/refresh (with refreshToken cookie)
    BE->>BE: Verify refresh token (JWT)
    BE->>DB: SELECT * FROM refresh_tokens WHERE token = ?
    DB-->>BE: Token record found
    BE->>BE: Generate new accessToken (JWT, 30m)
    BE-->>Interceptor: { accessToken }
    Interceptor->>Interceptor: Update stored token
    Interceptor->>BE: Retry original request with new accessToken
    BE-->>Client: Original response data (200 OK)

Sequence Diagram — Admin Creates Student

Sequence 
sequenceDiagram
    actor Admin as 👨‍💼 Admin
    participant FE as Admin Dashboard
    participant API as API Layer
    participant BE as Backend (admin.js)
    participant DB as MySQL DB

    Admin->>FE: Fill student creation form
    FE->>API: POST /admin/create_student
    API->>BE: { student data }
    BE->>BE: Validate fields
    BE->>BE: Hash password (bcrypt)

    BE->>DB: BEGIN TRANSACTION
    BE->>DB: INSERT INTO students (...)
    DB-->>BE: student_id

    BE->>DB: INSERT INTO student_fees (student_id, total_fee=85000)
    DB-->>BE: OK

    BE->>DB: SELECT * FROM courses WHERE department_id=? AND semester=?
    DB-->>BE: Matching courses list

    loop For each course
        BE->>DB: INSERT INTO enrollments (student_id, course_id, academic_year)
    end

    BE->>DB: COMMIT
    DB-->>BE: Transaction committed

    BE-->>API: { student_id }
    API-->>FE: Success response
    FE-->>Admin: Student created successfully

Sequence Diagram — Faculty Takes Attendance

Sequence 
sequenceDiagram
    actor Faculty as 👩‍🏫 Faculty
    participant FE as Faculty Dashboard
    participant API as API Layer
    participant BE as Backend (teacher.js)
    participant DB as MySQL DB

    Faculty->>FE: Select course for attendance
    FE->>API: POST /teacher/attendance/session
    API->>BE: { course_id, session_date }
    BE->>DB: INSERT INTO attendance_sessions (...)
    DB-->>BE: session_id
    BE-->>API: { session_id }
    API-->>FE: Session created

    FE->>API: GET /teacher/attendance/session/:session_id/students
    API->>BE: Request enrolled students
    BE->>DB: SELECT students enrolled in course (status=active)
    DB-->>BE: Student list
    BE-->>API: Student list
    API-->>FE: Display students

    Faculty->>FE: Mark attendance for each student
    FE->>API: POST /teacher/attendance/submit
    API->>BE: { session_id, attendance[] }

    loop For each student
        BE->>DB: INSERT INTO attendance_records (session_id, student_id, status)
    end

    DB-->>BE: All records inserted
    BE-->>API: Success
    API-->>FE: Attendance submitted
    FE-->>Faculty: Confirmation message

Activity Diagram — Student Promotion Flow

Activity 
flowchart TD
    Start([🟢 START]) --> SelectParams["Admin selects
department_id + current_semester
+ new_academic_year"]
    SelectParams --> FetchStudents["Fetch all active students
in dept + semester"]
    FetchStudents --> ForEach{"FOR EACH
student"}

    ForEach --> CheckSem{"semester == 8?"}

    CheckSem -->|Yes| Graduate["Mark status = 'graduated'"]
    Graduate --> NextStudent["NEXT student"]

    CheckSem -->|No| CalcNew["Calculate:
newSemester = current + 1
newYear = ceil(newSem / 2)"]
    CalcNew --> UpdateStudent["UPDATE student
(semester, year, academic_year)"]
    UpdateStudent --> SelectCourses["SELECT courses
for new semester"]
    SelectCourses --> InsertEnroll["INSERT enrollments
for each course"]
    InsertEnroll --> InsertFee["INSERT new fee record
(total_fee = 85000)"]
    InsertFee --> NextStudent

    NextStudent --> MoreStudents{"More students?"}
    MoreStudents -->|Yes| ForEach
    MoreStudents -->|No| Commit["COMMIT transaction"]
    Commit --> ReturnSuccess["Return success message"]
    ReturnSuccess --> End([🔴 END])
